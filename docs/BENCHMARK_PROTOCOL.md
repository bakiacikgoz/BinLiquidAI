# BENCHMARK_PROTOCOL

## Commands

```bash
uv run binliquid benchmark smoke --mode all --profile balanced
uv run binliquid benchmark ablation --mode all --profile balanced
uv run binliquid benchmark energy --profile balanced --energy-mode measured
```

## Ablation Modes

- `A`: LLM-only baseline
- `B`: LLM + rule router
- `C`: LLM + sLTC router
- `D`: LLM + sLTC router + memory writes

## Metrics

- `success_rate`
- `p50_latency_ms`
- `p95_latency_ms`
- `peak_ram_mb`
- `fallback_rate`
- `wrong_route_rate`
- `expert_call_rate`
- `memory_write_rate`
- `energy_estimate_wh`

## Energy Reporting

- `estimated`: always available model-based estimate
- `measured`: macOS `powermetrics` based; if permission is missing, deterministic detail is returned

## Result Files

- JSON results: `benchmarks/results/*.json`
- Markdown report: generated by `benchmark ablation`
